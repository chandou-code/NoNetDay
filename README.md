# 🔌 断网自控器 — 1 小时 Vibecoding 产物

> 从零到能跑，一个小时，纯 vibe 写的。

## 一句话简介

**24 小时断 / 24 小时通** 的「旧流量包模式」自控工具，本地网卡 + 路由器 MAC 过滤双重锁，DPAPI 加密配置防自己手贱改。

## 技术栈（吹一波）

| 层 | 技术 | 牛在哪 |
|---|---|---|
| 本地网卡控制 | `netsh interface set interface` + `subprocess` | 系统级禁用，不是改 DNS 那种小打小闹 |
| 路由器 MAC 过滤 | `requests` + 逆向华为光猫 Web 管理接口 | 不用装 App、不用 telnet，纯 HTTP 搞定，电信定制 E8-C 光猫实测可用 |
| 状态机调度 | 纯 Python 标准库 + `datetime` 滑动窗口 | 不依赖 schedule / APScheduler，冷启动零依赖额外库 |
| 配置加密 | **Windows DPAPI**（CryptProtectData） | 密钥由 Windows 管理，绑定用户 + 机器，代码里不存密钥，偷 config 文件也解不开 |
| 发布 | PyInstaller onefile 打包 | 单 exe 双击就跑，不用装 Python 环境 |

## 逆向光猫接口的几个亮点

1. **登录流程逆向**：先 `GetRandCount.asp` 拿随机 token，密码 base64 编码，再 `login.cgi` 提交 — 三次握手式登录，比普通 form 登录多一层随机数防重放。
2. **token 刷新机制**：每次请求后页面 `onttoken` 会变，必须从响应里重新提取，不然下一次请求直接 403。
3. **删除接口绕路方案**：`del.cgi` 要传选中状态参数太复杂，直接用 `set.cgi` 把目标 MAC 改成 `00:00:00:00:00:00` 占位，等效删除还能复用条目——**能用一行代码解决的绝不逆向 100 行 JS**。
4. **JS 变量提取**：MAC 列表存在 `MacFilter = new Array(...)` 的 JS 变量里，而且冒号是 `\x3a` 十六进制转义，正则一把梭直接抠出来。

## 工作原理

```
启动 → 读加密 config
   ↓
每隔 10 秒检查一次：距上次切换满 24 小时了吗？
   ↓
满了 → 切换状态
   ├─ 断网日：先路由器加 MAC → 再禁用本地网卡（顺序不能反！）
   └─ 联网日：先启用本地网卡 → 等网络就绪 → 再路由器删 MAC
   ↓
更新加密 config，继续循环
```

为什么顺序不能反？因为路由器是通过本地网卡访问的——你把网卡先禁了，还怎么调路由器接口？这个坑踩过一次就懂了。

## 用法（极简版）

```
1. 双击 断网调度器.exe（管理员身份）
2. 首次运行自动断网，24 小时后自动通，再 24 小时再断，循环往复
3. config.json 是加密的，用记事本打开是乱码，别白费力气了
```

要改周期 / MAC 地址 / 路由器密码？改源码重新打包，因为 DPAPI 加密 + 打包 exe 就是为了让你自己改不了——自控工具的核心就是「把钥匙交出去」。

## 安全边界（诚实版）

**能防**：
- 偷了 config 文件也解不开（DPAPI 绑定用户 + 机器）
- 随便改 exe 字节导致校验失败就跑不起来
- 路由器后台手动操作？可以，但你得记得路由器密码

**防不住**：
- 你有 admin 权限 → 理论上什么都能绕
- 你直接拔网线 / 关路由器 → 物理层攻击无解
- 你拿另一台手机开热点 → 社会工程学攻击无解
